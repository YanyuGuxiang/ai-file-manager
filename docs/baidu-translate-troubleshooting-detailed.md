# 百度翻译API故障排查指南

## 错误代码 54001: Invalid Sign 排查

当您看到错误代码 54001 时，这表示签名验证失败。虽然我们的本地测试显示签名算法是正确的，但问题可能存在于以下几个方面：

### 1. 检查百度翻译开放平台设置

#### 步骤1：登录百度翻译开放平台
1. 访问 https://api.fanyi.baidu.com/
2. 使用您的百度账号登录

#### 步骤2：检查应用状态
1. 进入"管理控制台"
2. 查找您的应用（APP ID: YOUR_ACTUAL_APP_ID）
3. 确认应用状态是否为"已启用"
4. 检查应用的"通用翻译API"服务是否已开通

#### 步骤3：核对密钥
1. 在应用详情页面，重新复制您的密钥
2. 与 .env 文件中的密钥进行对比
3. 确保没有额外的空格或字符
4. 注意密钥是区分大小写的

### 2. 账户状态检查

#### 实名认证
- 百度翻译API要求账户完成实名认证
- 前往百度账号中心完成实名认证

#### 账户余额
- 检查百度翻译API账户是否有足够的余额
- 如果账户欠费，可能会导致API调用失败

### 3. 验证凭证准确性

由于您的凭证格式看起来是正确的，我们可以尝试以下步骤：

1. 在百度翻译开放平台重新生成密钥：
   - 进入应用管理页面
   - 点击"重置密钥"
   - 使用新生成的密钥更新 .env 文件

2. 确保 .env 文件格式正确：
   ```
   BAIDU_TRANSLATE_APPID=your_actual_app_id_here
   BAIDU_TRANSLATE_SECRET=your_actual_secret_key_here
   ```

### 4. 临时测试验证

如果仍然遇到问题，您可以尝试使用在线MD5计算器手动验证签名：

1. 拼接字符串（以"test"为查询词为例）：
   `your_app_id_heretest123456your_secret_key_here`

2. 使用在线MD5工具生成哈希值
3. 确认生成的值是否与我们在调试脚本中得到的一致

### 5. 技术细节验证

根据我们的调试，拼接字符串为：
`your_app_id_heretest123456your_secret_key_here`

MD5哈希前的十六进制表示为：
`796f75725f6170705f69645f6865726574657374313233343536796f75725f7365637265745f6b65795f68657265`

生成的MD5签名应该是：
`e1a7ac80cf3aef756e1b00e27e50bab8`

如果您在百度开发者文档中看到不同的示例，请核对是否使用相同的格式。

### 6. 官方文档参考

参阅百度翻译API官方文档以确认请求格式：
https://api.fanyi.baidu.com/doc/21

### 7. 网络和IP设置

如果您在百度平台设置了IP白名单，请确保您的IP地址在允许列表中。

---

## 验证步骤总结

1. 重新登录百度翻译开放平台并检查账户状态
2. 核对APP ID和密钥的准确性
3. 确保完成了实名认证
4. 检查账户余额
5. 重新生成密钥（如果需要）
6. 更新 .env 文件
7. 运行测试脚本验证：`node test-baidu-translate.js`

如果完成以上所有步骤后仍然遇到问题，请联系百度翻译API技术支持以获得进一步帮助。
